# File Server - Курсовая работа

## Описание проекта(вариант задания)
2. File Server. Сервер предоставляет клиентам информационные ресурсы для скачивания 
(фильмы, фотографии, музыку и проч). Для получения доступа к ресурсам клиенту 
необходимо завести на сервере свой аккаунт и получить статус – простой пользователь, 
вип-пользователь. От статуса будет зависеть скорость скачивания. Каждая сессия 
клиента начинается с процедуры авторизации (ввод логина\пароля).

## Функциональность
- **Авторизация пользователей** по логину и паролю
- **Два уровня доступа**: обычный пользователь и VIP-пользователь
- **Дифференцированная скорость скачивания** в зависимости от статуса
- **Просмотр доступных файлов** на сервере
- **Скачивание файлов** с сервера на клиентское устройство

## Аутентификация
В системе реализована **авторизация по логину и паролю**. Пользователи создаются администратором сервера. Функционал самостоятельной регистрации через клиентское приложение не предусмотрен.

### Тестовые учётные записи:
| Логин | Пароль | Статус       |
|-------|--------|--------------|
| user  | 123    | Обычный пользователь |
| vip   | 123    | VIP-пользователь |

## Запуск приложения

### 1. Запуск сервера
```bash
cd src
javac server/MainServer.java
java server.MainServer
```

### 2. Запуск клиента
```bash
cd src
javac client/gui/MainClient.java
java client/gui/MainClient
```

## Особенности реализации
- Отсутствует функционал регистрации через клиент (пользователи создаются администратором)
- Скорость скачивания эмулируется через задержки в потоке передачи данных
- Пароли хранятся в хешированном виде
- Поддерживается многопользовательская работа
